Hermes Switch UI — MCP Server Lifecycle

Discover → Install → Activate → Handshake → Available → Use · capability-gated page

Discover → Install → Activate

  • • Discover via MCP Hub search (/api/mcp/hub-search) or manual entry
  • • Install writes config to ~/.hermes/mcp_servers/
  • • POST /api/mcp/configure or preset install
  • • Activate: toggle in detail drawer → gateway spins server up
  • • auth: none | bearer | oauth · toolMode: all | include | exclude

Handshake → Available

  • • MCP protocol: list tools, list prompts, list resources
  • • discoveredToolsCount populated on success
  • • status: connected — tools appear in agent context
  • • GET /api/mcp — server list with status indicators
  • • Test via POST /api/mcp/test → latency + tool count

Use + Error Paths

  • • Agent calls tool → MCP server executes → result back to agent
  • • Handshake fail → status: failed · lastError set
  • • Disconnect toggle → tools removed from context
  • • Retry path: re-enable → back to Activate stage
  • • Secrets: McpMaskedValue — never sent to client
