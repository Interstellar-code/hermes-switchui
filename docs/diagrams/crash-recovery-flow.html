Crash Recovery Flow

After a crash — identify what's broken → find which file owns it → apply the fix

File ownership at a glance

  • • Chat history → sessions.db
  • • Browser login sessions → workspace-sessions.json
  • • Gateway/dashboard URL overrides → workspace-overrides.json
  • • Theme + UI prefs → browser localStorage (claude-theme key)

Safe files to delete

  • • workspace-sessions.json — forces re-login, no chat data lost
  • • workspace-overrides.json — resets to HERMES_API_URL default
  • • Browser localStorage — resets theme and UI prefs only
  • • All recreated automatically on next startup

Risky files (backup first)

  • • sessions.db — deleting loses all chat history
  • • kanban.db — deleting loses all tasks and boards
  • • config.yaml — deleting requires provider reconfiguration
  • • .env — deleting requires re-entering all API keys
