Hermes Switch UI — Terminal PTY Architecture

How keystrokes travel from xterm.js in the browser to a real OS shell and back

Browser — xterm.js

  • • xterm.js renders VT100/ANSI output
  • • FitAddon resizes canvas to container
  • • Keystrokes sent via POST /api/terminal-input
  • • Output arrives via SSE /api/terminal-stream

Server — terminal-sessions.ts

  • • Session registry as in-memory Map
  • • Spawns pty-helper.py via child_process
  • • EventEmitter relays stdout to SSE listeners
  • • Idle reap timer (5 min default) cleans up

pty-helper.py — Python PTY bridge

  • • Uses os.openpty() for real TTY master/slave
  • • Avoids node-pty native addon requirement
  • • Handles TIOCSWINSZ for terminal resize
  • • Copied to dist/ at build time
