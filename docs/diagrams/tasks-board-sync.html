Task Board — dnd-kit + Agent Sync Flow

How drag-and-drop card moves propagate from UI to kanban.db and back via the Hermes gateway

TasksScreen

  • · Board / Swim / Timeline view toggle
  • · DndContext wraps all DroppableColumns
  • · DraggableCard wraps each TaskCard
  • · DragOverlay renders semi-transparent ghost
  • · New Task button opens TaskDialog

Optimistic update + POST

  • · onDragEnd triggers moveTask() locally first
  • · PATCH /api/hermes-kanban/tasks/:id
  • · column + position payload
  • · Rollback on fetch error
  • · Toast shown on failure

kanban.db + SSE broadcast

  • · Hermes gateway persists to SQLite kanban.db
  • · task:updated SSE event fires on change
  • · useKanbanEvents hook subscribes on mount
  • · queryClient.invalidateQueries re-fetches board
  • · Agent-side updates follow same broadcast path

Task Detail Drawer

  • · Opens on card click
  • · Inline edit: title, description, assignee, priority
  • · Archive: soft-delete (hidden from board)
  • · Hard delete: DELETE …?delete=true
  • · Status change available from drawer column picker
