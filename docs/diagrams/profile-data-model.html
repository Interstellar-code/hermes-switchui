Profile Data Model

Profile composition, on-disk storage layout, and how a chat session selects and loads a profile

Profile core

  • · Named configuration adopted at chat session start
  • · Identified by profile-id (slug)
  • · Stored under ~/.hermes/profiles/<profile-id>/
  • · config.yaml + memory/ subfolder

Persona + Memory

  • · Persona: name, system_prompt, tone
  • · Memory: seed *.md files pre-loaded into context
  • · Editable via Agent Memory tab in /memory page
  • · Memory files scoped per profile-id

Provider config

  • · provider: manifest | openai | …
  • · model: string (e.g. auto)
  • · key_env: references CUSTOM_API_KEY in ~/.hermes/.env
  • · base_url for custom OpenAI-compatible endpoints

Skills + MCP servers

  • · Skills: allowed skill IDs[] from marketplace or bundled
  • · MCP servers: allowed external tool server IDs[]
  • · Both configured in Profile wizard (step 6)
  • · Related docs: skills.md · mcp.md
