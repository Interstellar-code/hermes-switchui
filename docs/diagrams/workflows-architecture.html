Hermes Switch UI — Workflows Architecture

Frontend surfaces, backend engine, and storage — how data flows from library to live run

Frontend surfaces

  • • Library panel: browse bundled + user workflows
  • • Grid view: card tiles with tag filters
  • • Editor: Overview / DAG / YAML tabs
  • • Launch wizard: 4-step run submission
  • • Run detail panel: live node status + output

Backend engine

  • • Discovery loader: parses YAML, validates nodes
  • • DAG executor: topological execution
  • • Event emitter: node lifecycle events
  • • Node-runs projector: writes DB snapshot
  • • SSE relay: streams events to browser

Storage (~/.hermes/)

  • • workflows/ — user YAML definitions
  • • workflow-runs.db — run + node_runs tables
  • • Bundled definitions shipped with the app
  • • Manifest JSON for fast listing
