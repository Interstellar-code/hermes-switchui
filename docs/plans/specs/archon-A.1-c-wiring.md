## Goal

Port the cross-cutting modules that bind the engine together: the in-memory event emitter (SSE feed), the workflow router (chat→workflow translation), the dependency-injection bundle (`deps.ts`), small utility modules, and the bundled default workflow YAMLs.

This cluster does NOT touch the executor (A.1-a) or schemas/validators/discovery (A.1-b). Each file is a near-verbatim port with import-path rewrites.

## Adaptations per file

### `event-emitter.ts`
- Pure module. Imports event types from schemas — rewrite `from "./schemas"` → `from "../schemas"`.

### `router.ts`
- Imports schemas + types. Rewrite paths per the canonical map below.
- `@archon/paths` imports already aliased.

### `deps.ts`
- Imports `IWorkflowStore` — rewrite `from "./store"` → `from "./store-interface"` (sibling within `wiring/`).
- Imports `IAgentProvider` types from `@archon/providers/types` — already aliased.
- Imports `WorkflowEventEmitter` — rewrite `from "./event-emitter"` → `from "../emitter/event-emitter"`.

### `wiring/store-interface.ts`
- Copy ONLY the interface declarations from upstream `store.ts` (lines ~34-113). Strip any concrete implementation, helper classes, or instantiation code. Keep type exports, the `IWorkflowStore` interface, `WORKFLOW_EVENT_TYPES` constant array, and any inline type aliases used by the interface.
- The A.1.1 `SwitchUiWorkflowStore` already implements this interface — verify type-compatibility at the end (`tsc --noEmit` will catch mismatches).

### Utils
- All four are standalone or near-standalone. Imports limited to schemas + Node std. Rewrite paths.
- `variable-substitution.ts` is the SIMPLE substitution (`$VAR` literal). The DAG-aware substitution is in `executor-shared.ts` (A.1-a). Do NOT touch the executor-shared one.

### Defaults
- `bundled-defaults.generated.ts` is auto-generated by the upstream build (bundles 20 workflow YAMLs as TS string literals). Port verbatim — Switch UI ships these as `bundled` source for v1. v1.1 may regenerate at our build.
- `text-imports.d.ts` declares typing for `*.md` / `*.yaml` imports — keep as-is. May need a Vite plugin (`?raw`) later but **not for this spec**.
- `bundled-defaults.ts` is the thin loader that consumes `bundled-defaults.generated.ts`.

### `test-utils.ts`
- Port verbatim. Rewrite paths.

## Import-path rewrite map (canonical for this cluster)

```
./schemas               → ../schemas
./schemas/...           → ../schemas/...
./event-emitter         → ./event-emitter (within emitter/) OR ../emitter/event-emitter (elsewhere)
./router                → ./router (within routing/) OR ../routing/router (elsewhere)
./deps                  → ./deps (within wiring/) OR ../wiring/deps (elsewhere)
./store                 → ./store-interface (within wiring/) OR ../wiring/store-interface (elsewhere) for the interface only
./validator             → ../validation/validator
./validation-parser     → ../validation/validation-parser
./command-validation    → ../validation/command-validation
./loader                → ../discovery/loader
./workflow-discovery    → ../discovery/workflow-discovery
./script-discovery      → ../discovery/script-discovery
./executor              → ../core/executor
./executor-shared       → ../core/executor-shared
./dag-executor          → ../core/dag-executor
./condition-evaluator   → ../core/condition-evaluator
./logger                → ../core/logger
./hooks                 → ../core/hooks (if standalone file exists) OR ../schemas/hooks if schema-only
./utils/<x>             → ./utils/<x> if in utils/ folder OR ../utils/<x> elsewhere
./defaults/<x>          → ./defaults/<x> if in defaults/ OR ../defaults/<x> elsewhere
```

**No edits to `@archon/paths`, `@archon/git`, `@archon/providers/types`** — keep package names; aliases handle them.
